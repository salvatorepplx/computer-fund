# KILLED Artifacts

`runs/KILLED/<id>.json` holds proposals that were withdrawn before ever being ARMED — a PROPOSED
artifact that failed Computer's pre-promotion review under the CHARTER rails. A kill here is a
**win**, not a failure: it is the falsification machinery doing its job (per CONSTITUTION honest-KILL
ethos). No real money is ever placed from a KILLED artifact.

Each killed artifact preserves the original proposal payload verbatim and adds a top-level `killed`
block: `killed_at`, `killed_by`, the evidence-grounded `reason`, the `diagnostic` script that
produced the evidence, and `reopen_criteria` (the exact new evidence that would justify reconsidering
the idea). The narrative corpse also lands in `runs/CORPSES.md` and the reusable rule in
`corpus/lessons.md`.

This directory is offline/propose-only state. It does not authorize, size, or place any trade.
